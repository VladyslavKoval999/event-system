# Event Management Full-stack System

Це комплексна система управління подіями з аналітикою та підтримкою користувачів. Проєкт розроблено як підсумкову практичну роботу, що демонструє навички побудови REST/GraphQL API, авторизації, роботи з базою даних, налаштування WebSockets та візуалізації даних.

## 🚀 Основний функціонал

- **REST API:** Повноцінний CRUD для сутностей «Події» та «Учасники».
- **GraphQL API:** Гібридний підхід для гнучкого отримання складних вкладених структур (події разом з їхніми учасниками).
- **База даних:** Зберігання даних у хмарній базі MongoDB.
- **Безпека та Авторизація:** Захист маршрутів створення/редагування подій за допомогою сесій (`express-session`) та безпечне збереження паролів завдяки хешуванню (`bcrypt`).
- **Real-time Чат:** Чат підтримки в реальному часі без перезавантаження сторінки з використанням `Socket.io`.
- **Візуалізація даних:** Інтерактивний графік реєстрацій учасників за днями, побудований на `Recharts`.
- **Пагінація:** Курсорна пагінація (`limit/skip` та пошук за `_id`) для реалізації нескінченного скролу.

## 🛠 Технологічний стек

**Бекенд:**
- Node.js / Express.js
- MongoDB / Mongoose
- GraphQL / Apollo Server
- Socket.io
- bcrypt / express-session

**Фронтенд:**
- React (зібрано через Vite)
- React Router DOM
- Socket.io-client
- Recharts

## ⚙️ Встановлення та локальний запуск

### 1. Клонування репозиторію
Схиліть проєкт на свій комп'ютер:
`git clone <ваше-посилання-на-github-репозиторій>`

### 2. Налаштування Backend
Перейдіть у папку бекенду та встановіть необхідні залежності:
`cd Lab4`
`npm install`

Створіть файл `.env` у корені папки `Lab4` та додайте наступні змінні оточення:
`PORT=3000`
`MONGO_URI=ваше_посилання_на_кластер_mongodb`
`DB_NAME=events_db`
`SESSION_SECRET=секретний_ключ_для_сесій_12345`

Запустіть сервер:
`npm start`

### 3. Налаштування Frontend
Відкрийте новий термінал, перейдіть у папку фронтенду та встановіть залежності:
`cd frontend`
`npm install`

Запустіть клієнтську частину для розробки:
`npm run dev`

## ☁️ Деплоймент (Deployment)
Проєкт розгорнуто у хмарних сервісах:
- **Frontend (Клієнтська частина):** задеплоєно на платформі **Vercel**.
- **Backend (Серверна частина & API):** задеплоєно на платформі **Render**.
